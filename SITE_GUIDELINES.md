# SITE GUIDELINES

Этот файл нужен как локальная опора для дальнейшей работы по `YourWorld-Guide`.
Если в новом диалоге продолжать редактирование сайта, агент сможет прочитать этот файл и быстрее попасть в текущую систему без повторного долгого согласования.

## Цель сайта

- Сайт — это wiki и база знаний по проекту `YourWorld PVE`.
- Основной тон: понятный, игровой, аккуратный.
- Любая новая страница должна выглядеть как часть одной системы, а не как отдельный микросайт.

## Структура файлов

- Общий стиль сайта: [style.css](/c:/Users/Malina/Documents/GitHub/YourWorld-Guide/style.css)
- Локальные стили страниц: `styles/pages/<page>.css`
- Страница должна подключать общий `style.css` и при необходимости свой page-CSS.
- Большие `<style>` внутри HTML не использовать.

## Базовый каркас страницы

Для верхнеуровневых wiki-страниц использовать один и тот же порядок:

1. `page-top` с кнопкой назад
2. `hero` с иконкой страницы, заголовком и коротким описанием
3. основные секции через `card info-card`
4. `site-footer`

## Единый UI-паттерн

### Кнопки

- Для основных действий использовать один визуальный паттерн:
  - фон `rgba(255, 133, 0, 0.08)`
  - бордер `rgba(255, 170, 70, 0.24)`
  - hover `rgba(255, 133, 0, 0.14)`
  - радиус `12px`
  - высота `42px` или `44px`
- Кнопки одинакового смысла должны выглядеть одинаково:
  - `Открыть`
  - `Закрыть`
  - `Возможные награды`
  - кнопки-вкладки
  - кнопки-переходы

### Карточки

- Основные блоки делать через:
  - border `rgba(255,255,255,0.08)`
  - radius `24px`
  - мягкий тёмный фон с лёгким оранжевым glow
- Внутри карточек соблюдать ровный ритм:
  - заголовок
  - контент
  - пояснение / action

### Модалки

- Все модалки должны иметь:
  - затемнённый backdrop
  - скруглённую панель
  - кнопку `Закрыть` в том же стиле, что и action-кнопки
  - одинаковую логику закрытия по `Esc` и по клику вне панели

### Подписи к изображениям

- Если изображение можно увеличить, подпись должна быть короткой и одинаковой по смыслу:
  - `Нажмите на картинку, чтобы увеличить`
- Если изображение не увеличивается, не добавлять псевдо-подписи “для вида”.

## Текст и формулировки

- Писать по-русски, без канцелярщины и без лишней “технички”.
- Избегать разговорных форм в SEO и официальных описаниях:
  - плохо: `инфой`
  - хорошо: `информацией`
- Формулировки в пределах похожих разделов должны быть едины:
  - `Рекомендуемое оружие`
  - `Возможные награды`
  - `Необходимые предметы`
  - `Как зайти на сервер`

## SEO

Для каждой верхнеуровневой страницы должны быть:

- `title` в формате `Название | YourWorld Wiki`
- `meta description`
- `og:title`
- `og:description`
- `og:image`
- `og:type`
- `og:url`

Описание должно быть коротким, понятным и без сленга.

## Изображения и alt

- Все смысловые `img` должны иметь `alt`.
- `alt` должен описывать содержимое, а не повторять `image` или `photo`.
- Если картинка декоративная, можно оставлять пустой `alt=""`, но только если это действительно декор.

## Мобильная версия

- Верхние блоки и hero-секции обязаны нормально складываться в колонку.
- На мобильных:
  - уменьшать внутренние отступы
  - переводить сложные grid-блоки в 1 колонку
  - длинные кнопки растягивать на всю ширину
  - не оставлять слишком узкие фиксированные колонки
- Особое внимание:
  - главная
  - модалки
  - секции с карточками
  - длинные кнопочные панели

## Перед добавлением новой страницы

Нужно проверить:

- есть ли нормальный `hero`
- есть ли `page CSS` в `styles/pages/`
- есть ли мета-теги
- есть ли alt у изображений
- не выбивается ли страница по кнопкам, карточкам и отступам
- не ломается ли она на мобильной ширине

## Что не делать

- Не плодить новые визуальные паттерны без необходимости.
- Не вставлять большой локальный CSS прямо в HTML.
- Не делать кнопки одного смысла разными по цвету и форме.
- Не использовать разные стили заголовков и карточек на похожих страницах без причины.
